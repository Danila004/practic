# Отчёт о реализации фичи SCR-05: Детали записи и Отмена

## 1. Цель реализации

Реализация backend-части функционала просмотра деталей записи на тренировку и отмены записи с поддержкой стандартной и поздней отмены (менее чем за 10 минут до начала тренировки).

---

## 2. Покрытые требования

## 2.1. Функциональные требования (FR)

| ID | Название      | Описание |
|----|---------------|----------|
| FR-09 | Отмена записи | Реализован API endpoint `POST /bookings/{booking_id}/cancel`, который позволяет клиенту отменить активную запись. Backend проверяет статус записи и выполняет отмену только для записей со статусом `active`. |

---

## 2.2. Пользовательские сценарии (UC)

| ID | Название | Описание |
|----|----------|----------|
| UC-03 | Отмена записи клиентом | Клиент может отменить активную запись через экран деталей записи (SCR-05). При поздней отмене (менее 10 минут до начала) требуется дополнительное подтверждение. |
