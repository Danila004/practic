# Отчёт о реализации фичи SCR-04 «Мои записи»

## 1. Цель реализации

Реализовать экран «Мои записи» (SCR-04) мобильного приложения «Скалодром», предоставляющий клиенту возможность просматривать, группировать и управлять всеми своими бронированиями тренировок, включая активные, прошедшие и отменённые записи. Экран является центральным хабом для сценариев просмотра деталей брони (SCR-05), оценки инструктора (SCR-06) и получения уведомлений об отмене тренировки скалодромом (UC-05).

---

## 2. Покрытые требования

### 2.1. Функциональные требования (FR)

| ID | Требование | Описание реализации |
|---|---|---|
| FR-13 | Статус «Отменена скалодромом» с указанием причины | В ответе API возвращается поле `cancellationReason` из таблицы `training_slots`. Во frontend отображается красным текстом внутри карточки брони в секции «Отменённые». |

### 2.2. Пользовательские сценарии (UC)

| ID | Сценарий | Описание реализации |
|---|---|---|
| UC-03 | Просмотр и отмена записей | Tap по карточке брони выполняет `GET /booking/${bookingId}` — переход на SCR-05 (детали брони с возможностью отмены). |
