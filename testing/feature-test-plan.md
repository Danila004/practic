| C-10 | Экран записи (SCR-02) | Проверить, что если API возвращает `available_count = 0`, приложение отображает сообщение "Снаряжения для проката нет" и блокирует кнопку подтверждения записи. | Should have | UC-02 |
| C-11 | Экран записи (SCR-02) | Проверить, что при выполнении запроса на получение данных о прокате отображается индикатор загрузки, а кнопка подтверждения неактивна. | Should have | NFR-03 |
| C-12 | Экран подтверждения (SCR-03) | Проверить, что после успешного бронирования на экране подтверждения отображаются все детали: дата, время, формат, инструктор и выбранная опция снаряжения. | Must have | UC-02 |
| C-13 | Экран подтверждения (SCR-03) | Проверить, что кнопки "Вернуться к расписанию" и "В Мои записи" корректно выполняют переход на соответствующие экраны. | Must have | UC-02 |
| C-14 | Экран "Мои записи" (SCR-04) | Проверить, что записи сгруппированы по статусам (Активные, Прошедшие, Отмененные), а каждая запись имеет соответствующий цветовой индикатор: зеленый (active), серый (completed), желтый (cancelled_by_client), красный (cancelled_by_gym). | Must have | FR-13, UC-05 |
| C-15 | Экран "Мои записи" (SCR-04) | Проверить, что при наличии записей со статусом `cancelled_by_gym` поверх списка отображается баннер-уведомление с текстом "Одна из ваших тренировок была отменена..." | Must have | UC-05 |
| C-16 | Экран "Мои записи" (SCR-04) | Проверить, что при отсутствии броней отображается сообщение "У вас нет записей" с соответствующей иконкой. | Must have | - |
| C-17 | Экран "Мои записи" (SCR-04) | Проверить, что для записей со статусом `completed` отображается кнопка "Оценить", которая ведет на экран SCR-06. | Should have | UC-04 |
| C-18 | Экран деталей записи (SCR-05) | Проверить, что экран отображает все поля из объекта Booking: дату, время, зону, инструктора, статус, выбранную опцию снаряжения. | Must have | UC-03 |
| C-19 | Экран деталей записи (SCR-05) | Проверить, что для записи со статусом `cancelled_by_gym` на экране деталей отображается поле с причиной отмены, полученное из API. | Must have | FR-13, UC-05 |
| C-20 | Экран деталей записи (SCR-05) | Проверить, что кнопка "Отменить запись" (красная) отображается и активна только для записей со статусом `active` и скрыта для записей с другими статусами. | Should have | FR-09, UC-03 |
| C-21 | Экран деталей записи (SCR-05) | Проверить, что при переходе по несуществующему `booking_id` отображается сообщение "Запись не найдена". | Should have | UC-03 |
| C-22 | Экран оценки (SCR-06) | Проверить, что при наведении/выборе звезды обеспечивается визуальная обратная связь, и пользователь может выбрать только одну оценку (от 1 до 5 звезд). | Should have | UC-04 |
| C-23 | Экран оценки (SCR-06) | Проверить, что текстовое поле для комментария имеет ограничение максимальной длины в 500 символов и плейсхолдер "Оставьте комментарий (опционально)". | Should have | UC-04 |
| C-24 | Экран оценки (SCR-06) | Проверить, что при нажатии кнопки "Отправить оценку" формируется корректное тело запроса (stars, comment) и после успешного ответа отображается сообщение "Спасибо за вашу оценку!". | Should have | UC-04 |
| C-25 | Глобальный компонент ошибки (SCR-07) | Проверить, что при отсутствии сети и наличии кэша приложение отображает данные из кэша с плашкой внизу экрана с текстом "Данные обновлены [время]" и кнопкой "Обновить". | Could have | NFR-10, SCR-07 |
| C-26 | Глобальный компонент ошибки (SCR-07) | Проверить, что при отсутствии кэша и сети отображается полный экран ошибки с иконкой, текстом и кнопкой "Повторить попытку". | Must have | NFR-02, SCR-07 |
| C-27 | Глобальный компонент ошибки (SCR-07) | Проверить, что при нажатии "Повторить попытку" и повторных неудачах система выполняет повторные попытки через 3 секунды и 10 секунд, после чего показывает кнопку. | Should have | SCR-07 |
| C-28 | Модальное окно "Место занято" (SCR-08) | Проверить, что при ошибке `SLOT_FULL` отображается модальное окно с заголовком "К сожалению", текстом "К сожалению, все места заняты..." и кнопками "Выбрать другой слот" / "Понятно". | Must have | FR-08, UC-02 |
| C-29 | Модальное окно "Место занято" (SCR-08) | Проверить, что окно можно закрыть по крестику, свайпу вниз и по кнопкам, и при этом происходит возврат на SCR-01 без автоматического обновления списка. | Must have | UC-02 |
| C-30 | Навигация (Bottom Navigation Bar) | Проверить, что Bottom Navigation Bar содержит три пункта ("Расписание", "Мои записи", "Профиль") и корректно переключает между ними, подсвечивая активный экран. | Must have | UC-01, UC-03 |
| C-31 | Push-уведомления | Проверить, что при получении push-уведомления об отмене тренировки отображается корректный заголовок ("Тренировка отменена") и текст с указанием причины. | Must have | FR-15, UC-05 |
| C-32 | Push-уведомления | Проверить, что при получении push-напоминания за 2 часа до тренировки уведомление содержит время, формат, имя инструктора и адрес скалодрома. | Could have | FR-16, UC-06 |
| C-33 | Локализация | Проверить, что все текстовые элементы интерфейса (заголовки, кнопки, сообщения об ошибках) отображаются полностью на русском языке без "рыбы" и технических терминов. | Must have | NFR-07 |
| C-34 | Адаптивность | Проверить, что интерфейс корректно отображается на различных размерах экранов (маленькие, средние, большие смартфоны) в портретной и ландшафтной ориентациях. | Must have | NFR-01, FR-20 |
| C-35 | UX/UI | Проверить, что все интерактивные элементы (кнопки, карточки слотов) имеют целевые области не менее 44x44 pt для удобства нажатия на смартфонах. | Must have | NFR-08 |
| C-36 | Кэширование | Проверить, что при успешной загрузке данных они сохраняются в локальное хранилище, и при повторном открытии приложения в офлайн-режиме отображаются закэшированные данные. | Could have | NFR-10 |


# Сценарные тесты

| ID | Область | Описание | Приоритет | Источники |
|----|---------|----------|-----------|-----------|
| S-01 | Успешная запись на тренировку | **Полный позитивный сценарий (со своим снаряжением):** 1. Клиент видит список слотов на экране "Расписание". 2. Выбирает слот и переходит к выбору снаряжения. 3. Выбирает опцию "Со своим снаряжением". 4. Подтверждает запись. 5. Получает экран подтверждения с деталями. 6. Переходит в "Мои записи" и видит новую активную бронь. | Must have | BR-01, BR-02, UC-02 |
| S-02 | Успешная запись с прокатом снаряжения | **Полный позитивный сценарий (с прокатом):** 1. Клиент выбирает слот. 2. Выбирает опцию "Взять напрокат". 3. Приложение проверяет свободный прокатный фонд через API и отображает стоимость. 4. Клиент подтверждает запись. 5. Бронирование успешно создается. | Should have | BR-10, FR-05, FR-06, FR-07 |
| S-03 | Попытка записи при полном слоте | **Конкуренция за слот:** 1. Клиент инициирует запись на слот. 2. Бэкенд возвращает ошибку 409 SLOT_FULL. 3. Приложение отображает модальное окно SCR-08 с текстом "К сожалению, все места заняты...". 4. После нажатия "Выбрать другой слот" пользователь возвращается на SCR-01. Список не обновляется автоматически. | Must have | FR-08, UC-02, SCR-08 |
| S-04 | Попытка записи при полном прокатном фонде | **Недоступность снаряжения:** 1. Клиент выбирает слот. 2. Выбирает опцию "Взять напрокат". 3. API возвращает available_count = 0. 4. Приложение отображает сообщение "Снаряжения для проката нет". 5. Клиент переключается на опцию "Со своим снаряжением" и успешно записывается. | Should have | UC-02 |
| S-05 | Отмена записи (стандартная) | **Отмена за >10 минут до начала:** 1. Клиент открывает "Мои записи" и выбирает активную бронь. 2. Нажимает "Отменить запись". 3. Приложение отображает стандартный диалог подтверждения. 4. Клиент подтверждает. 5. Приложение отправляет запрос на отмену. 6. После успеха возвращает в "Мои записи", где статус брони меняется на cancelled_by_client. | Should have | BR-05, UC-03 |
| S-06 | Отмена записи (поздняя) | **Отмена за <10 минут до начала:** 1. Клиент пытается отменить бронь. 2. API возвращает 409 LATE_CANCELLATION_WARNING. 3. Приложение отображает диалог с предупреждением "Отмена менее чем за 10 минут до начала считается поздней...". 4. Клиент подтверждает повторно. 5. Приложение отправляет запрос с confirm_late_cancellation: true. 6. Запись отменена, статус обновлен. | Should have | FR-10, FR-11, UC-03 |
| S-07 | Отмена записи в диалоге подтверждения | **Отказ от отмены:** 1. Клиент нажимает "Отменить запись". 2. Приложение показывает диалог подтверждения. 3. Клиент нажимает "Отмена" в диалоге. 4. Диалог закрывается, запись остается активной, клиент возвращается на экран деталей. | Should have | UC-03 |
| S-08 | Отмена тренировки со стороны скалодрома | **Push-уведомление об отмене:** 1. Организатор отменяет слот в админ-панели. 2. Клиентское приложение получает push-уведомление с заголовком и причиной. 3. При открытии приложения статус брони в "Мои записи" отображается как "Отменена скалодромом" с указанием причины. 4. Слот в расписании недоступен для повторной записи. | Must have | BR-07, FR-13, FR-14, FR-15, UC-05 |
| S-09 | Отмена тренировки скалодромом (push не доставлен) | **Обработка недоставленного уведомления:** 1. Организатор отменяет слот. 2. Push-уведомление не доставлено (устройство офлайн). 3. Клиент открывает приложение позже. 4. Приложение выполняет запрос к GET /bookings и получает обновленный статус. 5. В "Мои записи" отображается баннер "Одна из ваших тренировок была отменена..." со статусом и причиной. | Must have | UC-05 |
| S-10 | Оценка инструктора после тренировки | **Сбор обратной связи:** 1. Тренировка завершается, статус брони становится completed. 2. Приложение отображает push-уведомление или баннер "Как прошла тренировка? Оцените инструктора!". 3. Клиент переходит к форме, выбирает 5 звезд, пишет комментарий "Отличная тренировка!" и отправляет. 4. Приложение получает подтверждение от API и показывает благодарность. | Should have | BR-06, FR-12, UC-04 |
| S-11 | Оценка инструктора (пропуск) | **Отказ от оценки:** 1. После завершения тренировки клиент получает предложение оценить инструктора. 2. В разделе "Мои записи" для данной записи остается доступной кнопка "Оценить" для повторного перехода. | Should have | UC-04 |
| S-12 | Оценка инструктора (ошибка API) | **Обработка ошибки при отправке:** 1. Клиент заполняет форму оценки и нажимает "Отправить". 2. API возвращает ошибку. 3. Приложение сохраняет оценку локально в виде черновика и предлагает повторную отправку при следующем открытии экрана. | Should have | UC-04 |
