# Use-cases

| UC-01 | **Актер:** Клиент.<br><br>**Предусловия:** Приложение установлено на мобильном устройстве. Клиент авторизован (при наличии системы аутентификации). Бэкенд доступен и отвечает на запросы.<br><br>**Основной поток:**<br>1. Клиент открывает экран расписания в приложении.<br>2. Приложение отправляет запрос к бэкенду на получение списка слотов на ближайшие 7 дней от текущей даты.<br>3. Бэкенд возвращает список доступных слотов с информацией по каждому: время, зона/формат, инструктор, свободные места.<br>4. Приложение отображает слоты в виде списка или календарной сетки.<br>5. Клиент просматривает доступные тренировки.<br>6. При отсутствии слотов в выбранном периоде приложение показывает empty state: «Пока нет доступных тренировок».<br><br>**Альтернативный поток (ошибка API):**<br>1. Бэкенд возвращает ошибку или не отвечает в течение таймаута.<br>2. Приложение отображает сообщение: «Не удалось загрузить расписание. Проверьте подключение и попробуйте снова».<br>3. Приложение предлагает кнопку для повторной попытки загрузки.<br><br>**Постусловия:** Клиент видит актуальное расписание на ближайшую неделю. Данные кэшированы для просмотра при отсутствии сети (при наличии реализации кэширования). | Must have | BR-03, FR-01, FR-03, FR-17, R-027, NFR-02 |
| UC-02 | **Актер:** Клиент.<br><br>**Предусловия:** Клиент находится на экране расписания. Выбран слот, у которого количество свободных мест больше нуля. Бэкенд доступен.<br><br>**Основной поток:**<br>1. Клиент нажимает кнопку «Записаться» на выбранном слоте.<br>2. Приложение отображает модальное окно с выбором опции снаряжения: «Со своим снаряжением» или «Взять напрокат».<br>3. Клиент выбирает один из вариантов.<br>4. При выборе «Взять напрокат» приложение:<br>&nbsp;&nbsp;4.1. Отображает стоимость проката согласно тарифу из бэкенда.<br>&nbsp;&nbsp;4.2. Проверяет наличие свободного прокатного фонда через API.<br>&nbsp;&nbsp;4.3. Если фонд пуст, отображает сообщение «Снаряжения для проката нет» и возвращает на шаг 2.<br>5. Приложение отправляет запрос на бронирование в бэкенд с указанием выбранной опции снаряжения.<br>6. Бэкенд проверяет наличие мест, резервирует слот и возвращает подтверждение бронирования.<br>7. Приложение отображает экран подтверждения с деталями записи: дата, время, формат, инструктор, выбранная опция снаряжения.<br><br>**Альтернативный поток (мест нет):**<br>1. На шаге 6 бэкенд возвращает отказ с причиной «Места закончились».<br>2. Приложение отображает сообщение: «К сожалению, все места заняты. Выберите другой слот».<br>3. Приложение возвращает клиента к списку слотов без перезагрузки.<br><br>**Альтернативный поток (ошибка API):**<br>1. Бэкенд возвращает ошибку или не отвечает.<br>2. Приложение отображает сообщение: «Не удалось выполнить запись. Попробуйте позже».<br>3. Состояние бронирования не изменяется.<br><br>**Постусловия:** Клиент успешно записан на тренировку. Слот отмечен как занятый в бэкенде. В списке «Мои записи» клиента появляется новая запись. В случае отказа — состояние системы не изменяется. | Must have | BR-01, BR-02, BR-04, BR-10, FR-04, FR-05, FR-06, FR-07, FR-08, R-015 |
| UC-03 | **Актер:** Клиент.<br><br>**Предусловия:** У клиента есть как минимум одна активная запись на будущую тренировку. Клиент авторизован.<br><br>**Основной поток:**<br>1. Клиент открывает раздел «Мои записи».<br>2. Приложение отображает список всех активных броней клиента.<br>3. Клиент выбирает конкретную запись для отмены.<br>4. Клиент нажимает кнопку «Отменить запись».<br>5. Приложение определяет временной интервал до начала тренировки.<br>6. Если до начала тренировки осталось более 10 минут:<br>&nbsp;&nbsp;6.1. Приложение отображает диалог подтверждения: «Вы уверены, что хотите отменить запись?».<br>&nbsp;&nbsp;6.2. Клиент подтверждает отмену.<br>&nbsp;&nbsp;6.3. Приложение отправляет запрос на отмену в бэкенд.<br>&nbsp;&nbsp;6.4. Бэкенд освобождает место.<br>&nbsp;&nbsp;6.5. Приложение отображает подтверждение: «Запись успешно отменена. Место освобождено».<br><br>**Альтернативный поток (менее 10 минут до начала):**<br>1. На шаге 6 приложение определяет, что до начала тренировки осталось менее 10 минут.<br>2. Приложение отображает предупреждение: «Отмена менее чем за 10 минут до начала считается поздней. Пожалуйста, подтвердите отмену».<br>3. Клиент подтверждает отмену.<br>4. Приложение отправляет запрос на отмену в бэкенд с маркером «поздняя отмена».<br>5. Бэкенд освобождает место и фиксирует факт поздней отмены.<br>6. Приложение отображает подтверждение отмены с уведомлением о том, что отмена была поздней.<br><br>**Постусловия:** Запись клиента отменена. Слот снова доступен для записи другими клиентами. В случае поздней отмены — факт зафиксирован в системе для аналитики и возможных ограничений (штрафов/лимитов) в будущем. | Should have | BR-05, FR-09, FR-10, FR-11 |
| UC-04 | **Актер:** Клиент.<br><br>**Предусловия:** Клиент имеет запись, которая перешла в статус «Состоялась» (тренировка завершена). Приложение получило от бэкенда уведомление о завершении тренировки.<br><br>**Основной поток:**<br>1. По завершении тренировки приложение получает push-уведомление или отображает баннер: «Как прошла тренировка? Оцените инструктора!».<br>2. Клиент нажимает на уведомление/баннер, открывается форма оценки.<br>3. Приложение отображает имя инструктора и звёздную шкалу оценки (от 1 до 5).<br>4. Клиент выбирает количество звёзд.<br>5. При желании клиент добавляет текстовый комментарий в текстовое поле.<br>6. Клиент нажимает кнопку «Отправить оценку».<br>7. Приложение отправляет данные оценки в бэкенд.<br>8. Бэкенд подтверждает сохранение.<br>9. Приложение отображает благодарность: «Спасибо за вашу оценку!».<br><br>**Альтернативный поток (пропуск оценки):**<br>1. Клиент закрывает форму оценки, не отправляя её.<br>2. Приложение сохраняет возможность оценить инструктора в разделе «Мои записи» — клиент может вернуться к оценке позже.<br><br>**Альтернативный поток (ошибка API):**<br>1. Бэкенд не подтверждает сохранение оценки.<br>2. Приложение отображает сообщение: «Не удалось отправить оценку. Попробуйте позже».<br>3. Приложение сохраняет черновик оценки локально и предлагает повторную отправку.<br><br>**Постусловия:** Оценка инструктора сохранена в бэкенде. Владелец скалодрома может видеть статистику оценок. Клиент вернулся на главный экран приложения. | Should have | BR-06, FR-12 |
| UC-05 | **Актер:** Система (бэкенд) по инициативе организатора / владельца скалодрома.<br><br>**Предусловия:** У клиента есть активная бронь на тренировку, которую организатор решил отменить (по причине профилактики, проверки снаряжения или иных обстоятельств). Инициация отмены происходит в существующей инфраструктуре (вне клиентского приложения).<br><br>**Основной поток:**<br>1. Организатор через существующую админку инициирует отмену тренировки (слота).<br>2. Бэкенд формирует событие отмены для всех клиентов, записанных на этот слот.<br>3. Клиентское приложение получает push-уведомление с заголовком «Тренировка отменена» и текстом: «Тренировка [дата, время] отменена скалодромом. Причина: [текст причины]. Приносим извинения за неудобства».<br>4. Приложение автоматически обновляет статус брони в локальном хранилище на «Отменена скалодромом» с указанием причины.<br>5. При следующем открытии раздела «Мои записи» приложение запрашивает актуальные данные с бэкенда и отображает бронь со статусом «Отменена скалодромом» и причиной.<br>6. Слот помечается как недоступный для повторной записи.<br><br>**Альтернативный поток (push-уведомление не доставлено):**<br>1. Клиент не получил push-уведомление (нет сети, отключены уведомления).<br>2. При следующем открытии приложения клиент видит обновлённый статус в «Моих записях».<br>3. Приложение отображает баннер: «Одна из ваших тренировок была отменена. Нажмите, чтобы узнать подробности».<br><br>**Постусловия:** Бронь клиента помечена статусом «Отменена скалодромом» с причиной. Слот недоступен для повторной записи. Клиент информирован об отмене (push или при следующем визите в приложение). | Must have | BR-07, FR-13, FR-14, FR-15 |
| UC-06 | **Актер:** Система (планировщик push-уведомлений).<br><br>**Предусловия:** У клиента есть активная запись на предстоящую тренировку. В системе настроено расписание отправки напоминаний (например, за 2 часа до начала). Клиент дал согласие на получение push-уведомлений.<br><br>**Основной поток:**<br>1. За установленное время до начала тренировки (например, за 2 часа) системный планировщик инициирует отправку напоминаний.<br>2. Система формирует список клиентов с активными бронями на ближайшие тренировки.<br>3. Для каждого клиента отправляется push-уведомление с текстом: «Напоминаем: сегодня в [время] тренировка [формат] с инструктором [имя]. Адрес: [адрес скалодрома]. Ждём вас!».<br>4. Клиент получает уведомление на мобильное устройство.<br><br>**Альтернативный поток (уведомление не доставлено):**<br>1. Push-сервис возвращает ошибку доставки (устройство офлайн, отключены уведомления).<br>2. Система фиксирует факт недоставки для аналитики.<br>3. Повторная попытка отправки не предпринимается (чтобы избежать дублирования при последующем подключении).<br><br>**Постусловия:** Клиент получил напоминание о предстоящей тренировке. Вероятность неявки снижена. | Could have | BR-08, FR-16 |
