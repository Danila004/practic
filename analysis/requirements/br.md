# Бизнес-требования

| ID | Формулировка | Приоритет | Ссылка на источник |
|----|--------------|-----------|---------------------|
| BR-01 | Автоматизация процесса записи клиентов на групповые тренировки для исключения ручного учёта и двойных броней | Must have | Описание проекта: ад с записью, путаница, двойные брони |
| BR-02 | Предоставление клиентам возможности самостоятельной записи на доступные слоты без участия администратора | Must have | Хочу нормальное приложение, чтобы люди сами записывались |
| BR-03 | Обеспечение видимости актуального расписания тренировок на неделю вперёд для клиентов | Must have | Расписание на неделю вперёд; клиент должен видеть свободные слоты |
| BR-04 | Управление лимитами наполняемости групп в зависимости от формата тренировки (16 чел. для опытных, 8 чел. для новичков) | Must have | Всего в группу до 16, но новичковые больше 8 не записываем |
| BR-05 | Предоставление возможности отмены записи клиентом с учётом временных ограничений | Should have | Люди отменяются. Если заранее — ок, если за 10 минут до начала — проблема |
| BR-06 | Сбор обратной связи от клиентов об инструкторах после тренировки | Should have | Хочу, чтобы после тренировки клиент мог поставить оценку инструктору |
| BR-07 | Информирование клиентов об отмене тренировки со стороны скалодрома (профилактика, проверка снаряжения) с указанием причины | Must have | Зону закрываем на профилактику, надо всех разом предупредить |
| BR-08 | Автоматическое напоминание клиентам о предстоящей записи для снижения числа неявок | Could have | Уведомления, чтобы напоминать про запись. А то забывают и не приходят |
| BR-09 | Возможность отметить постоянных клиентов и предоставлять им привилегии | Could have | Постоянных клиентов много, было бы здорово их как-то отмечать и давать что-то приятное |
| BR-10 | Обеспечение возможности учёта проката снаряжения при записи (со своими скальниками/страховкой или напрокат) | Should have | Выбирал, со своими скальниками и страховкой или ему дать напрокат |
