# Нефункциональные требования

| ID | Формулировка | Приоритет | Ссылка на источник |
|----|--------------|-----------|---------------------|
| NFR-01 | Приложение должно работать на мобильных устройствах (iOS и Android) с корректным отображением на различных размерах экранов | Must have | Пользоваться будут в основном с телефона — и клиенты, и инструкторы прямо в зале |
| NFR-02 | Приложение должно обеспечивать корректную обработку ошибок API с отображением понятных сообщений для пользователя | Must have | Приложение полагается на ответы бэкенда и корректно обрабатывает отказ |
| NFR-03 | Приложение должно асинхронно загружать данные для обеспечения плавного взаимодействия без блокировки интерфейса | Should have | Общее требование к UX мобильных приложений |
| NFR-04 | Приложение должно иметь время отклика интерфейса не более 2 секунд при стандартных операциях (загрузка расписания, запись, отмена) | Should have | Общее требование к производительности мобильных приложений |
| NFR-05 | Уведомления должны доставляться с минимальной задержкой (не более 5 минут с момента события) | Should have | Требование к оперативности оповещения об отмене тренировки |
| NFR-06 | Приложение должно работать в условиях нестабильного мобильного интернета (сохранять состояние, корректно обрабатывать таймауты) | Should have | Пользователи находятся в бывшем складском ангаре, покрытие может быть нестабильным |
| NFR-07 | Приложение должно поддерживать русскоязычный интерфейс полностью | Must have | Все пользователи — русскоязычные клиенты и персонал скалодрома |
| NFR-08 | Интерфейс приложения должен соответствовать принципам мобильного UX: крупные целевые области для нажатия, читаемый шрифт, минимальное количество шагов для выполнения ключевых операций | Must have | Общее требование к мобильным приложениям; клиенты всех возрастов |
| NFR-09 | Приложение должно быть разработано с учётом ограниченного бюджета: использовать кроссплатформенные подходы (React Native / Flutter) или нативные решения с минимальным дублированием кода | Must have | Бюджет ограниченный, давайте начнём с самого нужного |
| NFR-10 | Приложение должно поддерживать механизм кэширования данных для уменьшения количества запросов к API и обеспечения работы при слабом соединении | Could have | Пользователи в ангаре, возможны проблемы с интернетом |
| NFR-11 | Приложение должно логировать критические ошибки для возможности удалённой диагностики | Could have | Общее требование к надёжности и поддержке приложения |
| NFR-12 | Размер приложения не должен превышать 50 МБ для быстрой загрузки и установки на мобильные устройства | Should have | Общее требование к мобильным приложениям |
| NFR-13 | Приложение должно поддерживать корректную работу при одновременном использовании несколькими клиентами (конкуренция за слоты) — на уровне корректной обработки ответов бэкенда | Must have | Существующий бэкенд обеспечивает атомарную проверку свободных мест. Клиентское приложение корректно обрабатывает отказ |
