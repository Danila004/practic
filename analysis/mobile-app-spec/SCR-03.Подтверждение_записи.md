# SCR-03. Подтверждение записи (Success State)

## Техническая реализация

### 1. Отображение данных
- Иконка успеха (зелёная галочка)
- Детали записи из ответа `POST /bookings`:
  - Дата и время (из `slot.start_datetime`)
  - Формат (из `slot.zone.name`)
  - Инструктор (из `slot.instructor.full_name`)
  - Опция снаряжения (из `equipment_option`)

### 2. Кнопки навигации
- **"Вернуться к расписанию"** → переход на SCR-01
- **"В Мои записи"** → переход на SCR-04

### 3. Альтернативная реализация
- Если экран реализован как модальное окно:
  - Закрытие по нажатию крестика
  - Закрытие по свайпу вниз

### 4. Кэширование
- Данные о новой записи добавляются в локальный кэш SCR-04

### 5. API-зависимости
**Зависит от**: ответа `POST /bookings`

### 6. Покрытие требований
- **UC-02**: успешное бронирование
