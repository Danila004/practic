# SCR-05. Детали записи и Отмена

## Техническая реализация

### 1. Загрузка данных
**API Endpoint**: `GET /bookings/{booking_id}`
**Параметры**: `booking_id` из SCR-04

**Отображаемая информация**:
- Все поля из `Booking`
- Причина отмены (если `status = cancelled_by_gym`)

### 2. Кнопка отмены
- Отображается только для `status = active`
- Название: "Отменить запись"
- Цвет: красный

### 3. Диалог подтверждения

#### 3.1. Стандартная отмена
- Заголовок: "Подтверждение отмены"
- Текст: "Вы уверены, что хотите отменить запись?"
- Кнопки: "Отменить" / "Подтвердить"

#### 3.2. Поздняя отмена (<10 минут до начала)
- **API Endpoint**: `POST /bookings/{booking_id}/cancel`
- Ответ `409` с кодом `LATE_CANCELLATION_WARNING`
- Отображается диалог с предупреждением
- Текст: "Отмена менее чем за 10 минут до начала считается поздней. Пожалуйста, подтвердите отмену"
- Повторный запрос с параметром `confirm_late_cancellation: true`

### 4. Успешная отмена
- `200 OK`: обновление статуса на `cancelled_by_client`
- Возврат на SCR-04

### 5. Обработка ошибок
- `404`: показать сообщение "Запись не найдена"
- `500`: SCR-07

### 6. API-зависимости
**Зависит от**: `GET /bookings/{booking_id}`, `POST /bookings/{booking_id}/cancel`
**Использует**: `Booking`, `Error`

### 7. Покрытие требований
- **FR-09**: кнопка отмены
- **FR-10**: диалог подтверждения
- **FR-11**: предупреждение о поздней отмене
- **UC-03**: отмена записи
