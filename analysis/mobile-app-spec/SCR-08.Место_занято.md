# SCR-08. Место занято (Конкуренция за слот)

## Техническая реализация

### 1. Отображение модального окна
- Заголовок: "К сожалению"
- Текст: "К сожалению, все места заняты. Выберите другой слот"
- Иконка: предупреждение (жёлтый треугольник)

### 2. Действия
- Кнопка "Выбрать другой слот"
    - Закрытие модального окна
    - Возврат на SCR-01 (без перезагрузки списка)
- Кнопка "Понятно" (аналог)
    - Закрытие модального окна
    - Возврат на SCR-01

### 3. Закрытие
- Закрытие по крестику в правом верхнем углу
- Закрытие по свайпу вниз

### 4. Взаимодействие с API
**Код ошибки**: `SLOT_FULL`
**Источник**: ответ `409` от `POST /bookings`

### 5. Пользовательский сценарий
1. Пользователь выбрал слот
2. Нажал "Подтвердить запись"
3. Пока шла отправка, место занял другой пользователь
4. API вернул `409 SLOT_FULL`
5. Приложение показывает SCR-08
6. Пользователь возвращается к расписанию

### 6. Обновление данных
- При возврате на SCR-01 список **не обновляется** автоматически
- Пользователь должен выполнить Pull-to-Refresh или подождать 10 секунд (автообновление)

### 7. Покрытие требований
- **FR-08**: обработка ситуации "место занято"
- **UC-02**: альтернативный поток при недоступности слота
