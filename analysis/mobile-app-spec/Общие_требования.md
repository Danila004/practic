# Общие технические требования для мобильного приложения «Скалодром»

## 1. Платформа и технологии
- **Целевые платформы**: iOS 15.0+, Android 8.0+
- **Рекомендуемый фреймворк Frontend**: Flutter 3.x
- **Рекомендуемый фреймворк Backend**: Spring Boot 4.1.0
- **Язык Frontend**: Dart
- **Язык Backend**: Java

## 2. Сетевой слой
- **Базовый URL**: `http://localhost:8080`
- **Метод авторизации**: JWT Bearer Token в заголовке `Authorization`
- **Формат данных**: JSON
- **Таймаут запроса**: 10 секунд
- **Повторные попытки**: до 3-х попыток с экспоненциальной задержкой

## 3. Обработка ошибок
- **401 Unauthorized**: переход на экран авторизации (не входит в данный ТЗ)
- **409 Conflict**: показ модального окна с сообщением из поля `message`
- **500 Internal Server Error**: отображение экрана глобальной ошибки (SCR-07)
- **Network Error / Timeout**: экран офлайн-режима с кнопкой повторной попытки

## 4. Локальное кэширование
- **Расписание (SCR-01)**: кэшируется при успешной загрузке
- **Мои записи (SCR-04)**: кэшируются при успешной загрузке
- **Данные о снаряжении**: кэшируются на время сессии
- При отсутствии сети показываются кэшированные данные с отметкой времени последнего обновления

## 5. Навигация
- **Bottom Navigation Bar** с разделами:
    1. Расписание (SCR-01)
    2. Мои записи (SCR-04)
    3. Профиль (не входит в данный ТЗ)

## 6. Общие компоненты UI
- **Загрузчик**: индикатор прогресса (CircularProgressIndicator)
- **Empty State**: заглушка с иконкой и текстом
- **Модальные окна**: с возможностью закрытия по свайпу или крестику
- **Pull-to-Refresh**: на экранах со списками (SCR-01, SCR-04)

## 7. Соответствие требованиям
- Все функциональные требования (FR-01...FR-17) покрыты соответствующими экранами
- Все бизнес-требования (BR-01, BR-02, BR-04) реализованы через API
- Поддержка всех пользовательских сценариев (UC-01...UC-06)

## 8. Безопасность
- Хранение токена в защищенном хранилище Keychain / Keystore
- HTTPS для всех API-запросов(для разработки использовать HTTP)
- Валидация всех входных данных на клиенте перед отправкой
