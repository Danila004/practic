# SCR-07. Глобальная ошибка / Оффлайн-режим

## Техническая реализация

### 1. Состояния экрана

#### 1.1. Ошибка сети
- Иконка: "Нет соединения"
- Текст: "Не удалось загрузить данные. Проверьте подключение и попробуйте снова"
- Кнопка: "Повторить попытку"

#### 1.2. Ошибка API
- Иконка: "Ошибка сервера"
- Текст: "Что-то пошло не так. Попробуйте позже"
- Кнопка: "Повторить попытку"

### 2. Кэшированные данные
- Если в локальном кэше есть данные:
  - Отображается плашка внизу экрана
  - Текст: "Данные обновлены [время]"
  - Кнопка: "Обновить" (принудительно запросить API)

### 3. Повторная попытка

**Алгоритм повторных попыток**:
1. Первая попытка: немедленно
2. Вторая попытка: через 3 секунды
3. Третья попытка: через 10 секунд
4. При всех неудачах: показать кнопку "Повторить"

### 4. Обработка приоритетов
- **Приоритет 1**: показать кэшированные данные с плашкой ошибки
- **Приоритет 2**: если кэша нет, показать полный экран ошибки
- **Приоритет 3**: при восстановлении сети автоматически обновить данные

### 5. Таймаут операции
- **Общий таймаут**: 10 секунд
- При превышении → переход в офлайн-режим

### 6. Покрытие требований
- **UC-01**: повторная попытка загрузки
- **NFR-02**: устойчивость к ошибкам сети
- **NFR-10**: кэширование данных
