# SCR-04. Мои записи

## Техническая реализация

### 1. Загрузка данных
**API Endpoint**: `GET /bookings`
**Параметры**: без параметров (получаем все)

**Обработка ответа**:
- `200 OK`: отобразить список броней
- Пустой массив: показать Empty State

### 2. Группировка и сортировка
- **Сортировка**: по дате (от ближайшей к дальней)
- **Группировка по статусам**:
    1. Активные (status: `active`)
    2. Прошедшие (status: `completed`)
    3. Отменённые (status: `cancelled_by_client`, `cancelled_by_gym`)

### 3. Элемент списка
**Отображаемая информация**:
- Дата и время (из `slot.start_datetime`)
- Формат (из `slot.zone.name`)
- Инструктор (из `slot.instructor.full_name`)
- Статус с цветовой индикацией:
    - `active` → зелёный
    - `completed` → серый
    - `cancelled_by_client` → жёлтый
    - `cancelled_by_gym` → красный

**Действия**:
- Tap по записи → переход на SCR-05

### 4. Баннер-уведомление
- Отображается поверх списка при наличии `cancelled_by_gym`
- Текст: "Одна из ваших тренировок была отменена..."
- При нажатии → переход к SCR-05

### 5. Pull-to-Refresh
- Обновление списка с повторным запросом

### 6. Оценка инструктора
- Для записей со статусом `completed`
- Отображается кнопка "Оценить" → переход на SCR-06

### 7. Кэширование
- Сохранение последнего успешного ответа
- При офлайн-режиме показ кэшированных данных

### 8. API-зависимости
**Зависит от**: `GET /bookings`
**Использует**: `Booking`, `Slot`

### 9. Покрытие требований
- **FR-13**: статус "Отменена скалодромом" с причиной
- **UC-03**: просмотр и отмена записей
- **UC-04**: оценка инструктора
- **UC-05**: уведомления об отмене
