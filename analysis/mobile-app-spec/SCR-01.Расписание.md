# SCR-01. Расписание (Главный экран)

## Техническая реализация

### 1. Загрузка данных
**API Endpoint**: `GET /slots`
**Параметры запроса**:
- `from_date`: сегодня (по умолчанию)
- `to_date`: сегодня + 7 дней (по умолчанию)
- `zone_id`: опционально (для фильтрации)

**Обработка ответа**:
- `200 OK`: отобразить список слотов
- `200 OK` с пустым массивом: показать Empty State
- `500`: перейти к SCR-07

### 2. Компоненты экрана

#### 2.1. Адрес скалодрома
- Фиксированный текст: "FR-19"
- Отображается в верхней части экрана

#### 2.2. Список слотов
**Карточка слота содержит**:
- Время начала (формат: "HH:MM")
- Зона/формат тренировки (из `zone.name`)
- ФИО инструктора (из `instructor.full_name`)
- Количество свободных мест (из `available_spots`)

**Действия**:
- Tap по карточке → переход на SCR-02

#### 2.3. Фильтр/календарь
- Реализован как иконка календаря в AppBar
- При нажатии открывается DatePicker
- Выбранная дата передаётся в параметр `from_date`
- Список обновляется на 7 дней от выбранной даты

#### 2.4. Empty State
- Отображается при пустом массиве слотов
- Текст: "Пока нет доступных тренировок"
- Иконка: календарь с крестиком

### 3. Pull-to-Refresh
- Обновление списка с показом индикатора загрузки
- Повторный запрос к `GET /slots` с текущими параметрами

### 4. Навигация
- Bottom Navigation Bar: "Расписание" (активный), "Мои записи", "Профиль"
- Tap по "Мои записи" → переход на SCR-04

### 5. Обработка ошибок
- Таймаут/отсутствие сети → SCR-07
- 500 ошибка → SCR-07

### 6. Кэширование
- Сохранение последнего успешного ответа
- Отображение плашки: "Данные обновлены [время]"

### 7. API-зависимости
**Зависит от**: `GET /slots`
**Использует**: `ZoneInfo`, `InstructorInfo`

### 8. Покрытие требований
- **FR-01**: отображение слотов на 7 дней
- **FR-02**: фильтр/календарь для выбора дат
- **FR-03**: карточка слота с полной информацией
- **FR-17**: Empty State
- **UC-01**: просмотр расписания
